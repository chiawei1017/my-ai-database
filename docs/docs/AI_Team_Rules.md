## Development Rules

每次修改程式以前：

先閱讀：

Product_Blueprint.md

---

不得：

- 隨意修改架構
- 建立重複功能
- 建立重複檔案

---

每次完成工作：

必須說明：

1. 完成內容
2. 修改原因
3. 是否影響其它模組
4. 下一步建議

Coding Style

- 清楚

- 模組化
- 易維護

---

Principle

Think first.

Code second.
